## 3. Design Guidelines

### Layout
- Full-width sections with max-width `1280px` content area
- Alternating `#FFFFFF` and `#F6F8F5` (project background) section backgrounds
- Accent section backgrounds using `#EFF3ED` (background-secondary) or `#E8F1EC` (sidebar) for emphasis blocks
- Generous whitespace — 80px+ between sections
- 8px grid system (consistent with the product itself)

### Typography (Quicksand only — match the product exactly)
- **Font family**: `Quicksand` for ALL text — headings, body, CTAs, nav, footer. No secondary font.
- **Hero headings**: 700 weight, 48–64px, `text-primary` (`#1F2937`)
- **Section headings**: 600 weight, 36–48px
- **Sub-headings**: 500 weight, 24–32px
- **Body text**: 400 weight, 16–18px, line-height 1.75, `text-secondary` (`#4B5563`)
- **Muted/caption text**: 400 weight, 14px, `text-muted` (`#6B7280`)
- **CTA buttons**: 600 weight, 16px
- **Fallback stack**: `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`

### Color Usage Rules
- **Primary actions** (buttons, links, active states): `#1F3D36`
- **Primary hover**: `#17322C`
- **Accent highlights** (badges, decorative elements, feature icons): `#A7D7A2`
- **Page backgrounds**: alternate between `#FFFFFF` and `#F6F8F5`
- **Card backgrounds**: `#FFFFFF` with `shadow-card`
- **Elevated cards** (pricing, feature focus): `#FFFFFF` with `shadow-elevated` and `border: #E5E7EB`
- **Glow effects** on hover: `shadow-glow` (`0 0 20px rgba(31, 61, 54, 0.25)`)
- **Muted backgrounds** for badges/tags: `rgba(31, 61, 54, 0.1)`
- **Never use brown** — the project palette is green-based, not brown

### Buttons
- Primary: `bg-[#1F3D36]` text-white, rounded-xl, px-8 py-4, hover: `bg-[#17322C]`, transition 200ms
- Secondary: `border-[#1F3D36] text-[#1F3D36]`, rounded-xl, hover: `bg-[rgba(31,61,54,0.1)]`
- Ghost: `text-[#1F3D36]`, underline on hover
- Accent pill: `bg-[#A7D7A2] text-[#1F3D36]`, rounded-full, for tags and badges

### Mockups — iPhone 15 Pro & MacBook Frames
- **Customer app**: Always shown inside an **iPhone 15 Pro** frame (titanium finish, Dynamic Island visible)
- **Admin dashboard**: Always shown inside a **MacBook Pro** frame or a minimal browser window frame
- Use REAL product screenshots — not generic placeholder mockups
- **iPhone mockups** should be high-res with subtle drop shadow and slight 3D tilt (5–10° perspective)
- **MacBook mockups** can be flat or with a slight upward tilt
- For side-by-side hero: iPhone on the left (slightly overlapping), MacBook on the right
- For feature sections: alternate iPhone left / MacBook right across sections
- Transparent PNG mockup frames recommended (Clay Mockups 3D, Rotato, or Figma device frames)

### Animations — SaaS-Grade Motion Design

This is a premium SaaS product — the website must feel **alive, polished, and cutting-edge** without being distracting. Every section should have intentional motion.

**Global motion principles:**
- Easing: `cubic-bezier(0.16, 1, 0.3, 1)` (smooth deceleration) for entrances
- Duration: 600–800ms for section reveals, 200–300ms for micro-interactions
- Stagger: 100–150ms between sibling elements entering together
- Respect `prefers-reduced-motion` — disable all animation when user prefers reduced motion

**Hero section:**
- Headline: Word-by-word or line-by-line reveal with `clipPath` wipe from bottom, 800ms stagger
- Subheadline: Fade-up 200ms after headline completes
- CTA buttons: Scale from 0.95 → 1 + fade-in, 100ms stagger between primary and secondary
- iPhone mockup: Float-in from left with slight rotation correction (starts at `rotateY(8deg)` → `rotateY(0)`), 1000ms
- MacBook mockup: Float-in from right, mirror of iPhone animation
- Trust badges: Fade-up with 100ms stagger per badge, triggered 500ms after hero loads
- Background: Subtle animated gradient mesh or soft floating shapes using `#A7D7A2` at 5% opacity

**How It Works (3 Steps):**
- Connected timeline animation — as user scrolls, a progress line draws between steps 1 → 2 → 3
- Each step card: Scale-up from 0.9 + fade-in, triggered on scroll intersection (threshold 0.3)
- Step number: Counter morphs (1 → 2 → 3) with a circular progress ring in `#A7D7A2`
- iPhone mockup per step: Cross-fade between screenshots as the active step changes

**Feature grid cards:**
- Staggered entrance: Cards fly in from bottom with 100ms stagger, `translateY(40px)` → `0`
- Hover: Card lifts up 4px (`translateY(-4px)`) + `shadow-elevated` + subtle `#A7D7A2` border-glow
- Icon: Continuous slow pulse (scale 1 → 1.05 → 1, 3s loop) while in viewport

**Feature deep-dive sections (alternating left/right):**
- Scroll-triggered parallax: Mockup moves at 0.8x scroll speed, text at 1x (creates depth)
- Text block: Fade-up with `translateY(30px)` → `0`, 600ms
- Mockup (iPhone/MacBook): Slide-in from the side it's placed on + slight scale (0.95 → 1), 800ms
- Callout badges: Pop-in with spring physics (`spring: { stiffness: 300, damping: 20 }`), 150ms stagger
- Optional: Lottie animation or animated SVG icon per section for visual interest

**Comparison table:**
- Rows animate in with stagger (50ms per row) on scroll
- Checkmarks (✅): Scale-in with a satisfying pop (overshoot spring: 1 → 1.15 → 1)
- X marks (❌): Fade-in with subtle red flash
- Header row: Sticky on scroll with backdrop blur

**Pricing cards:**
- Cards: 3D tilt on hover using `perspective(1000px)` and `rotateX/Y` based on cursor position
- "Popular" badge: Gentle bounce animation (translateY 0 → -4px → 0, 2s loop)
- Price number: Count-up animation from 0 → actual price on scroll-enter
- Feature list: Stagger fade-in per line (50ms each)
- CTA button: Glow pulse on the recommended plan (`shadow-glow` fade in/out, 3s loop)

**Testimonials:**
- Auto-rotating carousel with cross-fade (5s per slide)
- Manual swipe/drag support on mobile
- Active card: Slight scale-up (1.02x) + elevated shadow
- Quote marks: Animated draw-in using SVG stroke-dashoffset

**Stats / counter section:**
- Numbers: Animated count-up (0 → target) using easeOut, triggered on scroll intersection
- Duration: 2s for count-up
- Suffix (+ or %) appears after count completes with a subtle bounce
- Decorative: Floating accent dots/circles in `#A7D7A2` at 10% opacity, slow drift animation

**CTA banner:**
- Full-width `#1F3D36` background with subtle animated noise/grain texture
- Text: Fade-up on scroll
- Buttons: Glow pulse effect on primary CTA
- Optional: Animated confetti burst in `#A7D7A2` particles on hover of the primary CTA

**Scroll progress:**
- Thin progress bar at top of page (`#1F3D36`, 3px height) that fills as user scrolls
- Smooth width transition

**Page transitions (if multi-page):**
- Cross-fade between pages (300ms)
- Or: Slide-up entrance for new page content

**Loading state:**
- Skeleton screens with shimmer animation (uses `#F0F4EE` → `#F6F8F5` gradient sweep)
- Logo pulse animation on initial load

**Cursor effects (desktop, optional):**
- Custom cursor ring that scales up on hovering interactive elements
- Magnetic effect: CTA buttons slightly pull toward cursor when within 50px

**Mobile-specific animations:**
- Reduce parallax to simple fade-in (parallax is janky on mobile)
- Keep stagger animations but reduce durations by 30%
- Swipe gestures for testimonial carousel
- Scroll-snap for feature sections on mobile

### Mobile Responsiveness
- Fully responsive — mobile-first approach
- Hamburger menu on mobile with slide-in drawer from right
- Stacked layouts for feature sections (mockup on top, text below)
- Full-width CTAs on mobile
- Touch-friendly spacing (min 48px tap targets)
- iPhone mockups scale down proportionally on mobile
- MacBook mockups switch to browser-frame style on mobile (no laptop bezel)
